# Прогнозирование цен на японские автомобили

Проект по машинному обучению для прогнозирования цен на японские автомобили на основе датасета с характеристиками транспортных средств.

## Описание задачи

Цель проекта — обучить модель предсказывать цены на японские автомобили, используя предоставленный датасет. Задача включает:
* подготовку данных;
* разделение на обучающую, тестовую и проверочную выборки;
* обучение нейронной сети;
* оценку качества модели через среднеквадратическую ошибку (MSE) и расчёт среднего процента ошибки на проверочной выборке.

**Критерии оценки:**
* 3 балла — выполнение задания;
* 4 балла — ошибка менее 10 %;
* 5 балла — ошибка менее 5 %.

## Датасет

Датасет содержит информацию о японских автомобилях, включая:
* марку и модель;
* год выпуска;
* пробег;
* объём двигателя;
* тип трансмиссии;
* привод;
* тип топлива;
* цену.

**Источник:** датасет доступен на [Kaggle](https://www.kaggle.com) и может быть загружен по ссылке:

```bash
wget https://storage.yandexcloud.net/academy.ai/japan_cars_dataset.csv
```

## Структура данных

После предварительной обработки датасет имеет следующие столбцы:

    -price — цена автомобиля (целевая переменная);

    -mark — марка автомобиля;

    -model — модель автомобиля;

    -year — год выпуска;

    -mileage — пробег;

    -engine_capacity — объём двигателя;

    -transmission — тип трансмиссии;

    -drive — привод;

    -fuel — тип топлива.

Удалены столбцы Unnamed: 0 и hand_drive.
## Используемые технологии

    -Python;

    -pandas — для работы с данными;

    -NumPy — для математических операций;

    -scikit‑learn — для разделения данных и предобработки;

    -TensorFlow/Keras — для построения и обучения нейронной сети;

    -matplotlib — для визуализации результатов.

## Процесс работы

    Загрузка и очистка данных:

        загрузка датасета из указанного источника;

        удаление строк с пропущенными значениями.

    Предобработка данных:

        кодирование категориальных переменных (mark, model, transmission, drive, fuel) с помощью LabelEncoder;

        нормализация целевой переменной (price) с помощью MinMaxScaler;

        масштабирование признаков с помощью StandardScaler.

    Разделение данных:

        разделение на обучающую (train), проверочную (validation) и тестовую (test) выборки (соотношение 60 : 20 : 20).

    Построение модели:

        архитектура: полносвязная нейронная сеть с BatchNormalization и Dropout;

        оптимизатор: RMSprop;

        функция потерь: MSE;

        метрика: MAE.

    Обучение модели:

        количество эпох: 200;

        сохранение лучшей модели с помощью ModelCheckpoint.

    Оценка качества:

        расчёт MSE и MAE на тестовой выборке;

        определение среднего процента ошибки.

## Архитектура модели

Модель состоит из следующих слоёв:

    -Dense (20 нейронов, ReLU) + BatchNormalization + Dropout (0.2);

    -Dense (200 нейронов, ReLU) + BatchNormalization + Dropout (0.5);

    -Dense (1 нейрон, линейный активатор).

## Результаты

В ходе обучения модель демонстрирует постепенное снижение потерь на обучающей и проверочной выборках. Конечное качество оценивается по:

    -MSE на тестовой выборке;

    -среднему проценту ошибки (рассчитывается на основе предсказаний и реальных значений цен).
